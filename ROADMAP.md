# Roadmap

`adas-perception` v0 is intentionally small: it draws lane candidates, road user detections, traffic sign and traffic light candidates, simple track IDs, rough monocular distances, and saves annotated images or videos. The current BDD100K path includes COCO-pretrained YOLO baselines, an ensemble baseline, a single YOLOv8n model fine-tuned on BDD100K road objects with tuned per-kind score thresholds, hard-frame/small-object sampling experiments, even/odd split 1024px retraining experiments, object-crop and copy-paste augmentation experiments, inference-time Test-Time Augmentation (horizontal flip + multi-scale via Ultralytics `augment=True`) with retuned thresholds, tiny-size-bucket threshold overrides, cached threshold sweeps, Weighted Box Fusion over multiple inference settings (no-TTA + TTA@1024 + tl-only retrain + TTA@960/1280/1536/1792) with per-kind IoU thresholds, and TP/FP/FN error analysis reports. The single-config accuracy-priority baseline is 1024px kind-tuned + TTA + tiny-override at `macro F1=0.6389` on the odd-index 5,000 report split. The accuracy ceiling is the 7-source WBF fusion with per-kind IoU thresholds, now available both as an offline cache workflow at `macro F1=0.6753` and as a single-config online pipeline (`configs/bdd100k_yolo_wbf7_perkind_iou_online.yaml`, `objects.fusion.mode=wbf`) at `macro F1=0.6763` and `FPS=3.44` on RTX 4070. That is +0.0408 over the no-TTA `0.6355` baseline, with all four kinds well above previous: `pedestrian=0.647 (+0.053)`, `traffic_light=0.632 (+0.049)`, `traffic_sign=0.648 (+0.035)`, `vehicle=0.779 (+0.027)`. Adding imgsz=2048 as an 8th source only adds `+0.0001`, so 1792 is where scale diversity plateaus; adding more retraining sources beyond tl-only also hurts because scale diversity already saturates the ensemble. Retraining-based augmentation runs alone still stay below `0.6355`.

## Near Term

- Move from val-frame split evaluation to official train/val or train/val/test style evaluation once the full public dataset is available locally. The YOLO exporter already supports separate `--val-images-root` and `--val-labels`, so the remaining blocker is local official train data.
- Extend per-kind score threshold tuning by scene, weather, and time-of-day groups.
- Retrain YOLO from the official BDD100K train split once the full public dataset is available locally. The validation-mirror even/odd 1024px 1epoch retraining run landed at `macro F1=0.6309` on odd 5,000 frames, below the `0.6355` 1024px kind-tuned baseline, so it should be treated as a comparison run rather than the default.
- Improve class imbalance handling beyond simple frame repeats. The first class-aware oversampling run landed at `macro F1=0.6332` on odd 5,000 frames and improved traffic light F1, but missed the current `0.6355` baseline. Full TP/FP/FN analysis shows frame repeats reduce traffic light/sign FP but also lose tiny/small recall, while pedestrian FP grows.
- Refine small-object crop training. The first object-crop 1024px/1epoch run dropped to `macro F1=0.5938`, and threshold tuning only recovered to `0.5943`, so crop-only oversampling should not be used as the default.
- Refine copy-paste augmentation. The first full-frame copy-paste 1024px/1epoch run reached `macro F1=0.6223`, and cached threshold tuning recovered to `0.6298`; reducing the added copy-paste set to about 1,000 images reached `0.6295` untuned and `0.6280` after cached threshold tuning. Feather-blended 2,500-image copy-paste reached `0.6234` untuned and `0.6242` after cached threshold tuning, so simple edge blending does not fix the distribution gap. This is much better than crop-only and can slightly improve traffic light F1, but it is still below the `0.6355` baseline. Next tests should use segmentation-mask-like pasted regions, filter pasted objects more carefully, and move to the official train split.
- Add crop-ratio caps so rare small targets can be mixed into varied full-frame road contexts without overtraining on crop-only views.
- Re-run hard-frame/small-object sampling with longer training, adjusted augmentation, and copy-factor sweeps. The first 1epoch experiment and its threshold tuning did not beat the split-tuned baseline.
- Build on the 1024px kind-tuned result with smarter post-processing and training changes. 1024px with traffic sign threshold retuned to 0.25 is the high-speed demo baseline. Conservative tiny-tile and tiny-recall size-aware threshold experiments slightly improved some size buckets but lost macro F1 due to false positives.
- Explore inference-side augmentation further. 1024px kind-tuned + Ultralytics TTA (`augment=True`) + per-kind threshold re-tuning reached `macro F1=0.6385` on odd 5,000 frames with `FPS=25.69`, beating the previous `0.6355`/`29.15 FPS` baseline without retraining. Adding a tiny-bucket threshold override (`traffic_light=0.25`, `traffic_sign=0.30` for the tiny bucket; non-tiny buckets stay at the kind threshold) reached `macro F1=0.6389` at `FPS=24.71`, with all four kinds non-negative vs the previous baseline. Tested follow-ups that did not help: combining TTA with the traffic_light-only copy-paste retrain (tune 0.6363, report 0.6319) and a finer 0.025-step threshold grid (tune 0.6478 but report 0.6384, i.e., tune-split overfit).
- Push further with Weighted Box Fusion across multiple inference settings. The ladder (all macro F1 on odd-index 5,000 report split): 2-way (no-TTA + TTA@1024) `0.6447`, 3-way (+ tl-only retrain TTA) `0.6489`, 4-way (+ TTA@1280) `0.6602`, 5-way (+ TTA@960) `0.6627`, 6-way (+ TTA@1536) `0.6686`, 7-way (+ TTA@1792) `0.6724`, 7-way with uniform `iou_thr=0.45` `0.6747`, 7-way with per-kind `iou_thr` (pedestrian/vehicle 0.50, traffic_light/traffic_sign 0.40) `0.6753`, 8-way (+ TTA@2048) `0.6725` (plateau), 8-way with class-balanced/pedestrian-only retrain `0.6717`/`0.6709` (ensemble saturated). Scale diversity via different imgsz values is the biggest driver (4-way `+0.0113` is the single largest step), and extreme scales with less overlap against the 1024-TTA multi-scale range matter more than nearby scales. Adding a second retraining source (v2-mask / class-balanced / ped-only) beyond tl-only hurts, showing that ensemble source selection needs care and retraining diversity saturates after the first complementary source. Relaxing the fusion `iou_thr` below the default 0.55 helps because dense small objects (traffic lights/signs) benefit from looser clustering, and making the IoU threshold per-kind (looser for small/dense classes, tighter for large/sparse classes like vehicles) matches that asymmetry. 7-way with per-kind `iou_thr` is the offline accuracy ceiling (~2.8 FPS, 7 forward passes). The single-config online accuracy setting stays TTA tuned + tiny override at `macro F1=0.6389`.
- Reduce dense-scene label collisions in visualization.
- Improve lane robustness with optional segmentation-based lane models.
- Replace heuristic sign labels with a two-stage sign detector/classifier.
- Replace heuristic traffic light colors with a trained light detector/state classifier.
- Upgrade simple IoU tracking to a motion-aware tracker.

## Next Capabilities

- Better monocular distance estimation using camera calibration and object-size priors.
- Traffic light housing detection and stronger state classification.
- More JSON-based evaluation metrics and comparison reports.
- Public dataset adapters beyond BDD100K, such as TuSimple/CULane for lanes and Mapillary/Tsinghua-Tencent style sign datasets where licensing allows.
- Markdown/CSV benchmark and evaluation comparison reports across devices, models, and configs.
- Visualization cleanup for dense scenes, including label collision reduction and optional class filters.

## Non-Goals for v0

- Full autonomous driving behavior planning.
- Vehicle control.
- Claims of safety suitability.
- Heavy plugin frameworks or large service infrastructure.
